## Overview

`maps.py` defines the four Pydantic data models that represent everything in a loaded map. These are the objects that flow through the whole program.

| Class        | Role                                                    |
| ------------ | ------------------------------------------------------- |
| `Drone`      | A single drone with a name and current position         |
| `Hub`        | A node on the map with coordinates and metadata         |
| `Connection` | A bidirectional link between two hubs                   |
| `Map`        | The full map — owns hubs, connections, and drone fleet  |

---

## `Drone`

```python
class Drone(BaseModel):
    name: str
    position: tuple[int, int] | None = None
    start_pos: tuple[int, int] | None = None
```

Represents one drone. Created automatically by `Map`'s model validator — you never instantiate `Drone` directly.

### Validator — `prepare_drone`

```python
@model_validator(mode='after')
def prepare_drone(self):
    self.name = "Drone " + self.name
    return self
```

Runs after the model is created. Prepends `"Drone "` to whatever name was passed (e.g. `"0"` becomes `"Drone 0"`).

---

## `Hub`

```python
class Hub(BaseModel):
    name: str = Field(min_length=1)
    x: int = Field(default=0)
    y: int = Field(default=0)
    metadata: dict[str, str]
```

One node on the map. `metadata` holds arbitrary key-value pairs parsed from the map file (e.g. `max_drones`, `zone`, `hub_type`).

### Known metadata keys

| Key        | Set by         | Meaning                                            |
| ---------- | -------------- | -------------------------------------------------- |
| `hub_type` | `FileHandler`  | `"start"` or `"end"` — identifies source and sink |
| `max_drones` | map file     | Max drones that can occupy this hub simultaneously |
| `zone`     | map file       | `"blocked"`, `"priority"`, or `"restricted"`       |

> `hub_type` is injected by `FileHandler.read_map_file()`, not read from the file itself — it's derived from the line prefix (`start_hub:` / `end_hub:`).

---

## `Connection`

```python
class Connection(BaseModel):
    from_hub: str
    to_hub: str
    metadata: dict[str, str]
```

A link between two hubs. Stored as directional (`from_hub` → `to_hub`) but treated as **bidirectional** by the pathfinder — edges are added in both directions when building the flow graph.

### Known metadata keys

| Key                 | Meaning                                            |
| ------------------- | -------------------------------------------------- |
| `max_link_capacity` | Max drones that can use this link in one tick      |

---

## `Map`

```python
class Map(BaseModel):
    hub_list: list[Hub]
    nb_drones: int
    name: str = Field(default='Mistery Map')
    difficulty: str = Field(default='Secret difficulty')
    drone_list: list[Drone] = Field(default_factory=list)
    connections: list[Connection] = Field(default_factory=list)
```

The top-level container. Holds everything needed to run the pathfinder and renderer.

### Fields

| Field        | Type               | Description                                    |
| ------------ | ------------------ | ---------------------------------------------- |
| `hub_list`   | `list[Hub]`        | All hubs parsed from the map file              |
| `nb_drones`  | `int`              | Number of drones to route                      |
| `name`       | `str`              | Display name (from `#` comment on line 1)      |
| `difficulty` | `str`              | Difficulty label (from `#` comment on line 1)  |
| `drone_list` | `list[Drone]`      | Auto-generated by the model validator          |
| `connections`| `list[Connection]` | All connections parsed from the map file       |

### Validator — `generate_drones`

```python
@model_validator(mode='after')
def generate_drones(self):
    start_hub = next(
        (h for h in self.hub_list
         if h.metadata.get('hub_type') == 'start'), None)
    start_pos = (start_hub.x, start_hub.y) if start_hub else None
    self.drone_list = [
        Drone(name=str(i), position=start_pos, start_pos=start_pos)
        for i in range(self.nb_drones)]
    return self
```

Runs after all fields are validated. Finds the start hub, then creates `nb_drones` `Drone` objects all positioned at the start hub's coordinates.

> `@model_validator(mode='after')` runs once the entire model is ready, so `self.hub_list` and `self.nb_drones` are both available.

### `map_info()`

Prints a summary of the map to stdout — name, difficulty, drone count, and each hub's details.
